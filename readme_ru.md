# Monogatari

Monogatari — открытая AI-ориентированная среда создания и исполнения визуальных новелл. Она сочетает свободную ролевую игру внутри сцены с детерминированными правилами сюжета: сгенерированный диалог остаётся живым, но модель не получает прямого контроля над маршрутами, очками и концовками.

> **Состояние проекта:** v0.9.5 активно разрабатывается. Основные контуры авторинга, исполнения, проверки, Web/PWA и упаковки для Windows уже реализованы; до версии 1.0 интерфейсы и контракты данных могут меняться.

## Ключевой принцип

Каждый интерактивный ход разделён на три зоны ответственности:

1. **Генератор NPC** создаёт только видимый ответ от лица персонажа.
2. Независимый **оценщик** предлагает структурированные изменения очков и доказательств.
3. **Детерминированный автомат** проверяет предложения и единолично выбирает переход или концовку.

Эта граница делает сюжетную логику проверяемой, защищает авторский канон и позволяет воспроизводить важные маршруты без поставщика модели.

## Возможности

- Узлы ролевой игры с целями, Knowledge, очками, доказательствами, бюджетами, переходами, тайм-аутами и концовками.
- Визуальные редакторы персонажей, Knowledge, сцен, диалогов, процессов, событий, концовок, звука и настроек.
- Защита от prompt injection, ограниченный контекст, проверка привязки к миру и авторские сценарии восстановления.
- Воспроизводимые Quality Suites для маршрутов, стабильности персонажей, границ и враждебного ввода.
- Web/PWA с локальным WebGPU-инференсом и офлайн-ресурсами проекта.
- Пакеты Windows с совместимыми локальными средами или проверенными OpenAI-совместимыми сервисами.
- Live2D, 2D-спрайты, GLB/GLTF 3D и многоступенчатый резервный рендеринг.
- Проверяемые пакеты `.monogatari` с переносимыми путями и реестрами SHA-256.
- MCP и Skill репозитория для контролируемого авторинга с помощью Agent.

## Поток одного хода

```text
Свободный ввод игрока
          │
          ├─ граница безопасности и привязки к миру
          ▼
 генератор ответа NPC ──► видимый диалог
          │
          ▼
 независимый оценщик ──► предложения очков/доказательств
          │
          ▼
 детерминированный автомат ──► следующий узел / концовка
```

## Быстрый старт

Требуются Node.js 20+, Rust stable и платформенные зависимости Tauri 2. WebGPU нужен только для тестирования локального инференса в браузере.

```bash
git clone https://github.com/SakalioLabs/Monogatari.git
cd Monogatari/frontend
npm install
npm run dev
```

Приложение Windows:

```bash
cd Monogatari/rust-engine/crates/tauri-app
cargo tauri dev
```

Проверка:

```bash
node scripts/verify-modules.mjs --list
node scripts/verify-modules.mjs
node scripts/verify-release.mjs
```

## Структура репозитория

| Путь | Назначение |
|---|---|
| `frontend/` | Интерфейс Vue 3 + TypeScript, Playtest и Web/PWA |
| `rust-engine/` | Среда Rust, граница авторинга, MCP и Tauri |
| `data/` | Канонический встроенный проект и исполняемые проверки |
| `docs/` | Архитектура, формат данных, пакеты, MCP и выпуск |
| `.agents/skills/` | Процесс Agent-авторинга для Monogatari |
| `scripts/` | Проверка модулей, ресурсов, зеркал, пакетов и релиза |

Начните с [`docs/DATA_FORMAT.md`](docs/DATA_FORMAT.md). Интеграции Agent должны использовать документированный [MCP-сервер](docs/MCP_SERVER.md) и транзакции с точными предусловиями. Секреты нельзя хранить в данных проекта.
